Click to Edit — Description

  1. On page load, the profile is rendered in display mode (read-only).
  2. Clicking Edit sends a GET to /click-to-edit/edit — the server responds with a patchElements SSE that swaps #demo with the edit form, initialising signals (firstName, lastName, email) with the current profile values.
  3. Clicking Save sends a PUT to /click-to-edit with the current signals (firstName, lastName, email) in the request body — the server persists the data and patches #demo back to display mode.
  4. Clicking Cancel sends a GET to /click-to-edit/cancel — discards changes and patches back to display mode.
  5. Clicking Reset sends a PATCH to /click-to-edit/reset — restores the original profile.
