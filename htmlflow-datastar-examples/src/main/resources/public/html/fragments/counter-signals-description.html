Counter (Signals) — Description

  1. On page load, a count signal is initialised to 0 via data-signals. A GET to /counter-signals/events opens an SSE stream that patches the count signal via patchSignals whenever the counter changes.
  2. Clicking + sends a POST to /counter-signals/increment — the server increments the counter and the SSE stream emits the updated value.
  3. Clicking − sends a POST to /counter-signals/decrement — the server decrements the counter and the SSE stream emits the updated value.
  4. The counter display updates automatically via data-text bound to the count signal — no DOM patching needed.
