Delete Row — Description

  1. On page load, the user table is rendered server-side with a Delete button per row and a Reset button.
  2. Clicking Delete on a row sends a DELETE to /delete-row/{index} — the server removes the row from #row-{index} via patchElements with mode = Remove.
  3. Clicking Reset sends a PATCH to /delete-row/reset — the server clears the deleted indices and patches #users-table with the full table via patchElements with mode = Replace.
  4. All buttons are disabled during any in-flight request via the _fetching indicator. The Delete button also requires a browser confirmation before firing.
