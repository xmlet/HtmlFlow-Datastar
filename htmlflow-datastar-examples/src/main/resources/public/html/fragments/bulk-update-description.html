Bulk Update — Description

  1. On page load, the user list is rendered server-side with a selections signal initialized.
  2. The header checkbox uses data-effect to reflect the state of all selections, and setAll to toggle all checkboxes at once.
  3. Clicking Activate sends a PUT to /bulk-update/activate with the selections array in the request body — the server updates the selected users to Active, resets selections to all false via patchSignals, and patches <tbody> via patchElements.
  4. Clicking Deactivate sends a PUT to /bulk-update/deactivate with the same payload — the server updates the selected users to Inactive and responds identically.
