Click to Load — Description

  1. On page load, signals offset = 0 and limit = 5 are initialised via data-signals. The table starts empty.
  2. Clicking Load More sends a GET to /click-to-load/more with offset and limit as query parameters — the server simulates a delay, generates the next batch of agents, updates offset via patchSignals, and appends the new rows to <tbody> via patchElements with mode = Append.
  3. The button is disabled while the request is in-flight via the _fetching indicator, and its label switches to Loading... via data-text.
