File Upload — Description

  1. On page load, a files signal is initialised via data-signals. The file input binds to it via data-bind.
  2. Clicking Submit sends a POST to /file-upload with the files signal (name, contents as Base64, MIME type) in the request body — the server validates each file against the 1MB limit, logs invalid files via executeScript, and patches #file-upload with a table of valid files via patchElements with mode = Replace.
  3. The button is disabled via aria-disabled when no files are selected.
