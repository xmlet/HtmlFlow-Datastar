Click to Edit (Signals) — Description

  1. On page load, signals (firstName, lastName, email) are initialised via data-signals. A GET to /click-to-edit-signals/events opens an SSE stream that patches the signals with the current server state.
  2. Clicking Edit toggles the form into view — no server request.
  3. Clicking Save sends a PUT to /click-to-edit-signals with the current signals — the server persists and emits them back via SSE.
  4. Clicking Cancel sends a GET to /click-to-edit-signals/cancel — the server re-emits the current state, reverting any unsaved changes.
  5. Clicking Reset sends a PATCH to /click-to-edit-signals/reset — the server restores the default profile and emits it via SSE.
